==================================================
ADDITIONAL PRODUCT RULES
========================

==================================================
CARRY-OVER SYSTEM
=================

Any unfinished target automatically carries over to the next day.

Example:

June 12

☑ DSA Problems

☐ Workout

June 13

☐ Workout (carried over)

Carried-over targets remain active until completed.

The original creation date should be preserved for history and analytics.

==================================================
NO-TASK DAY BEHAVIOR
====================

If a user creates zero targets on a day:

* The streak does NOT increase.
* The streak does NOT decrease.
* The day is considered neutral.

Example:

Current Streak = 10

Day with 0 targets

Current Streak remains = 10

This prevents users from losing streaks simply because they chose not to plan tasks that day.

==================================================
FRIEND REQUEST RESTRICTIONS
===========================

Users cannot:

* Send a friend request to themselves.
* Send duplicate friend requests.
* Send a friend request to an existing friend.
* Send another request while a request is already pending.

Show appropriate error messages when these actions are attempted.

==================================================
NOTIFICATION SYSTEM
===================

Notifications should support:

* Read status
* Unread status

Unread notifications should display a badge count.

Example:

🔔 Notifications (3)

When a notification is opened or marked as read:

* Remove it from the unread count.
* Preserve it in notification history.

==================================================
TARGET MANAGEMENT
=================

Users can:

* Create targets
* Edit target titles
* Delete targets
* Mark targets complete
* Mark completed targets incomplete

Deleting a target should update:

* Today's completion percentage
* Daily Score
* Points calculations
* Streak calculations (if affected)

Editing a target should only modify the title and should not affect statistics.
